    Manifests are an application definition standard for Varvara software.

    Metadata

    If you want to supply metadata to your rom, the address of that metadata must be placed at location 0101 in its source code (which ends up at location 0001 in the assembled ROM). This will allow for an external program to navigate the binary data and locate the metadata.

    If supported, writing the metadata's address to the #f0 device, via #f0 DEO, informs the emulator of the metadata's location. And emulator may choose to handle this information to name the window, or display an icon when a ROM is started.

    |0100
	;meta #f0 DEO

    A Varvara ROM is able to store certain information about itself, such as:

    • name: application name
    • version: release id
    • description: what it is
    • author: who built it
    • icon: a 64x64 pixels monochromatic graphic

    The metadata format begins with the address of the end of the metadata block, followed by the address of the appicon (an 8x8 icon), and finally four zero-terminated strings:

    @meta =&end =appicon
	( name ) "Catclock $1
	( version ) "Ver. 20 "4 $1
	( details ) "tic 20 "tac 20 "cat 20 "clock $1
	( author ) "Hundred 20 "Rabbits 20 7f 20 "2022 $2
	&end

    Manifest

    A manifest is a format for allowing Varvara applications to specify what menubar options it provides.

    A manifest.tal is a support library to this end. You can see support coverage here, it is also used in the definition of applications in Potato.

    @manifest
	04 "Application $1
		01 'n :file-new "New $1
		01 'r :file-rename "Rename $1
		01 'o :file-open "Open $1
		01 's :file-save "Save $1
	03 "Edit $1
		01 'c :edit-copy "Copy $1
		01 'v :edit-paste "Paste $1
		01 'x :edit-cut "Cut $1
	02 "Select $1
		00 1b :select-reset "Reset $1
		01 'a :select-all "All $1
	$1
